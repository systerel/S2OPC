Detailed Description

Non blocking fifo based on SOPC_DoubleBuffer mechanism, used by SOPC_RT_Subscriber to access received messages with several modes (simple FIFO, pop latest message, get latest message)

Implement a shared variable between one writer and several readers. Access mode can be:

  • normal : value stored into an internal fifo
  • new last : only the most up to date value is read and removed from fifo for the concerned reader
  • last : only the most up to date value is read but not removed from fifo for the concerned reader See SOPC_MsgBox and SOPC_MsgBox_DataHandle description to see how to use API.

Message box handle.

To use message box, follow the steps above:

  • 1)Create message box with SOPC_MsgBox_Create
  • 2)From writer, use SOPC_MsgBox_Push to add event, or use SOPC_MsgBox_DataHandle_XXX functions to work directly on

internal buffer

  • 3)From reader, initialize POP operation with SOPC_MsgBox_Pop_Initialize
  • 4)From reader, get event pointer with SOPC_MsgBox_Pop_GetEvtPtr
  • 5)From reader, release event point with SOPC_MsgBox_Pop_Finalize

)

Message box pop operation. Used to get an event.

This function shall be used AFTER a Pop_Initialize.

Multiple calls can be performed.

After all calls, SOPC_MsgBox_Pop_Finalize shall be called.

Parameters
[in]pMsgBoxMessage box object
[in]idBufferIdentifier returned by SOPC_MsgBox_Pop_Initialize
[in]idclientClient identifier
[out]ppDataPointer on data of an event. Shall not be freed or written.
[out]pSizeSize of data event returned
[out]pNbPendOrIgnoreEventsNumber of pending events
[in]modeMode GET_NORMAL, GET_LAST or GET_NEW_LAST
  • If mode GET_NORMAL, reader pop a new event from its point of view.
  • If mode GET_NEW_LAST, reader pop the last event if it is new from its point of view.
  • If mode GET_LAST, reader pop the last event even if not new (read several time the same last event)
Returns
SOPC_STATUS_OK if data is returned via ppData. SOPC_INVALID_STATE if API is in use concurrently by for same client id.
