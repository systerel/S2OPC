SOPC_ReturnStatus SOPC_ToolkitConfig_SetCircularLogPath ( const char *  logDirPath,
bool  createDirectory 
)

Configure the toolkit log generation properties (SOPC_Toolkit_Initialize required, !SOPC_Toolkit_Configured required)

Parameters
logDirPathAbsolute or relative path of the directory to be used for logs (full path shall exists or createDirectory flag shall be set, path shall terminate with directory separator). Default value is execution directory (same * as "" value provided).
createDirectoryFlag indicating if the directory (last item of path regarding directory separator) shall be created
Warning
The value of the pointer logDirPath is used afterwards by the Toolkit. The string is not copied. Hence, it must not be modified nor freed by the caller before SOPC_Toolkit_Clear.
SOPC_ReturnStatus SOPC_ToolkitConfig_SetCircularLogProperties ( uint32_t  maxBytes,
uint16_t  maxFiles 
)

Configure the toolkit log generation properties (SOPC_Toolkit_Initialize required, !SOPC_Toolkit_Configured required)

Parameters
maxBytesA maximum amount of bytes (> 100) by log file before opening a new file incrementing the integer suffix. It is a best effort value (amount verified after each print). Default value is 1048576.
maxFilesA maximum number of files (> 0) to be used, when reached the older log file is overwritten (starting with *_00001.log). Default value is 50.
