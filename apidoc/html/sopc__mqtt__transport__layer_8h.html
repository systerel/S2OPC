◆ SOPC_MQTT_TRANSPORT_SYNCH_GetHandle()

MqttTransportHandle* SOPC_MQTT_TRANSPORT_SYNCH_GetHandle ( MqttManagerHandle *  pWks,
const char *  uri,
const char *  topicName,
FctMessageSyncReceived  getMsg,
void *  pUserContext 
)

Return a new transport synchrone handle.

Note that the returned handle, 1 by new broker connection, is scheduled by the same mqtt manager. So, callback passed as parameter for 2 client is thread safe between 2 clients.

Parameters
pWksMQTT Manager handle
uriUri of broker
topicNameMQTT topic name used for subscriptions
getMsgCallback of message reception. If NULL, SOPC_MQTT_TRANSPORT_SYNCH_ReadMessage must be used to retrieve SOPC_Buffer data message.
pUserContextPassed when calling getMsg
