Detailed Description

File Transfer API.

Note
The file is saved locally as a ByteString
The MAX_SIZE_FILE value defines the maximum size of a file
Two callbacks are provided to the user (in opening and closing method). Its allows the user to save the file on their own. The user can also limit the opening mode.
Limitations :
  • Allows only one simultaneous opening per file.
  • A file handle remains valid on close session (standard OPC UA deviation).
  • UserWritable variable are not supported (initialize to true at the first open method call, when Writable variable is not defined)

◆ SOPC_FileTransfer_UserOpen_Callback

typedef void SOPC_FileTransfer_UserOpen_Callback(const SOPC_OpenMode openMode, bool *authorized, SOPC_ByteString *fileByteString)

User callback for the file opening processing. Allows the user to limit the opening mode. Allows the user to synchronize the contents of a file (for example : in the event of modification without using read/write methods of FileTransfer).

Parameters
openModeA Byte representing the opening mode requested by the client on the Open method.
[out]authorizedA Boolean pointer returned by the user to allow him to limit authorization of the openMode received by the Open method.
[out]fileByteStringA Bytestring pointer representing the file contents. if NULL, takes the contents of the locally saved file. otherwise, takes fileByteString as the new file contents.
Note
S2OPC File Transfer module provides a valid pointer to a SOPC_ByteString for fileByteString. If needed, the user must allocate the contents of fileByteString. It will be released by S2OPC. The size of fileByteString must be less than MAX_SIZE_FILE.
Warning
No blocking or long treatment in this callback.

◆ SOPC_FileTransfer_UserClose_Callback

typedef void SOPC_FileTransfer_UserClose_Callback(SOPC_ByteString *fileByteString, bool *localSave)

User callback for the file closing processing. Allows the user to save the file on their own.

Parameters
fileByteStringA Bytestring pointer representing the fileType contents.
[out]localSaveA Boolean pointer returned by the user to indicate if the fileByteString need to be store locally. 0 save, 1 erase.
Note
the fileByteString memory management is handled by the S2OPC. If the user wants to keep the fileByteString, he must copy it.
Warning
No blocking or long treatment in this callback.
